# CivicFlow Database Architecture

CivicFlow uses **Firebase Firestore** as its primary NoSQL database. This document defines the schema, collections, and versioning strategies used to maintain the event-driven analytics architecture.

---

## 1. Collection: `tickets`

**Purpose**: Stores all civic issues reported by citizens. This is the core transactional collection of the platform.

### Field Definitions
- `id` (String, Auto-generated ID): Unique identifier.
- `timestamp` (String, ISO 8601): When the ticket was created.
- `location` (Map):
  - `lat` (Float)
  - `lng` (Float)
- `description` (String): Original text from the user.
- `image_url` (String): Base64 encoded string or Firebase Storage URL.
- `upvotes` (Integer): Number of community verifications.
- `status` (String): Current status ("Open", "In Progress", "Resolved").
- `ai_assessment` (Map): Agent 1 (Assessor) output.
  - `category` (String)
  - `severity` (String)
  - `summary` (String)
- `routing` (Map): Agent 2 (Router) output.
  - `department` (String)
  - `priority` (String)
  - `recommended_action` (String)

### Indexes
- Single-field indexes on `status` and `routing.department` for dashboard filtering.
- Composite index on `timestamp` (desc) to fetch recent tickets.

---

## 2. Collection: `analytics_metadata`

**Purpose**: Tracks the global state of the dataset and manages optimistic locking to coordinate the background analytics workers. Contains exactly one document: `state`.

### Document: `state`
- `datasetVersion` (Integer): Incremented whenever a ticket is added or modified. Represents the current "truth" of the database.
- `analyticsVersion` (Integer): The version of the dataset that the currently cached analytics were built against.
- `stale` (Boolean): Derived flag. True if `datasetVersion > analyticsVersion`.
- `lastAnalyticsRun` (String, ISO 8601): Timestamp of the last successful Agent 3 generation.
- `ticketCount` (Integer): Total number of tickets in the database.
- `generationInProgress` (Boolean): Optimistic lock flag. Set to true when a background worker is currently executing Gemini.

---

## 3. Collection: `analytics_cache`

**Purpose**: Stores the pre-computed predictive insights generated by Agent 3 (Analyst) and the Python backend. Contains exactly one document: `latest`.

### Document: `latest`
- `generatedAt` (String, ISO 8601): When this cache was created.
- `generatedForVersion` (Integer): Matches the `datasetVersion` it was built against.
- `executiveSummary` (String): LLM-generated high-level overview.
- `riskClusters` (Array of Maps): LLM-identified hotspots.
  - `cluster_name` (String)
  - `description` (String)
  - `risk_level` (String)
- `recommendations` (Array of Maps): LLM-generated preventative actions.
  - `department` (String)
  - `action` (String)
- `heatmapData` (Array of Maps): Python-derived geospatial data.
  - `lat` (Float)
  - `lng` (Float)
  - `weight` (Integer)
- `departmentStatistics` (Map): Python-derived counts per department.
- `categoryStatistics` (Map): Python-derived counts per category.

---

## 4. Versioning & Locking Strategy

Because Firestore does not natively support long-running, cross-collection atomic transactions that encompass external API calls (like calling Gemini), CivicFlow uses an **Event-Driven Optimistic Locking** strategy.

1. **Invalidation**: A new ticket is created. The backend atomically increments `datasetVersion` and sets `stale = true` in `analytics_metadata/state`.
2. **Lock Acquisition**: A background worker reads `analytics_metadata`. If `generationInProgress` is false and `stale` is true, it sets `generationInProgress = true` within a Firestore transaction.
3. **Execution**: The worker safely takes ~10 seconds to fetch tickets and call Gemini.
4. **Commit**: The worker writes the new data to `analytics_cache/latest`, updates `analyticsVersion` to match the target `datasetVersion`, and sets `generationInProgress = false`.

If another ticket arrives during Step 3, `datasetVersion` increments again. The worker will finish its run, but the `analyticsVersion` will immediately be lower than the new `datasetVersion`, leaving `stale = true` and allowing the next worker to run immediately.
